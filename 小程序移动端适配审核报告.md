## 二、当前仍存在的优化建议（按优先级排序）

### P0 — 必须修复的功能问题

#### 1. 教师端「学生管理」页面半屏弹窗（.dialog）在华为小屏手机上可能被截断
- **文件**：`students.vue` 第 107-141 行
- **问题**：`.dialog` 固定宽度 `620rpx`（≈ 82%屏宽），含 canvas 雷达图、多行信息，华为 P40 等小屏手机（360px 视口）可能内容溢出
- **建议**：改 `width: 620rpx` → `width: 86%; max-width: 640rpx`；给 `.dialog` 加 `max-height: 80vh; overflow-y: auto;`

#### 2. 批量导入学生时，文件选择后需要班级上下文确认
- **文件**：`students.vue` 第 475-490 行
- **问题**：`commit()` 虽已加自动选班级逻辑，但如果用户从未进入某个班级页面就直接点学生 tab，班级选择可能不准确
- **建议**：在导入面板上方显示当前选中的班级名称，提供切换班级的能力

#### 3. 学校管理员「功能配置」全屏页面在低端机上加载慢
- **文件**：`school-admin.vue` 第 78-103 行
- **问题**：16 项功能的 checkbox 列表一次性渲染，页面无骨架/loading
- **建议**：加 `v-if="featUser"` + 打开时显示 `loading` 态

### P1 — 显著提升体验

#### 4. 所有表单提交成功/失败后自动清空/恢复状态
- **涉及**：`school-admin.vue` saveForm、`admin.vue` saveSchool、`students.vue` save 等
- **问题**：部分表单保存成功后未重置 `phoneError` 等校验状态
- **建议**：在每个 `save()` 的 finally 中统一重置校验状态 `phoneError.value = ''`

#### 5. 超管后台 Tab 切换数据不自动刷新
- **文件**：`admin.vue`
- **问题**：从「学校管理」切到「学校管理员」时，不重新拉数据（需手动触发 loadAdmins）
- **建议**：在 tab 切换时（`watch(tab, ...)`）重新 `loadAll()`

#### 6. 学校管理员列表分页/加载更多
- **文件**：`school-admin.vue`
- **问题**：教师列表一次性拉全量，学校规模大时体验差（当前已有 `createdAt DESC` 排序但无分页）
- **建议**：后端增加 `skip/take` 参数，前端加「加载更多」

#### 7. 微信登录获取昵称失败时的 fallback
- **文件**：`login.vue` doWechatLogin
- **问题**：`uni.getUserProfile` 在部分版本的小程序可能被拒或返回空，此时 `wechatNickName` 为空，教师默认名称为空
- **建议**：后端 `bindByNumber` 中当 nickName 为空时，使用 `'老师' + teacherNo 后4位` 作为默认名

### P2 — 建议优化

#### 8. 统一空状态提示图标风格
- **涉及**：`classes.vue`、`students.vue`、`school-admin.vue`、`admin.vue`
- **现状**：部分用 EmptyState 组件，部分用纯文本 `.empty`
- **建议**：统一使用 EmptyState 组件（含 icon + text + hint）

#### 9. 底部留白不足（全面屏手势误触）
- **文件**：大部分列表页面
- **问题**：部分列表页底部与原生 tabBar 间留白不够，最后一条内容可能被手势区遮挡
- **建议**：每个列表页容器加 `padding-bottom: calc(20rpx + env(safe-area-inset-bottom))`

#### 10. 暗色模式部分页面未完整适配
- **文件**：`school-admin.vue`、部分详情弹窗
- **问题**：`.dialog` 弹窗背景颜色在暗色模式下可能未正确继承 `--c-card`
- **建议**：统一检查所有 `.dialog`、`.sheet`、`.mask` 使用 CSS 变量

#### 11. loading/toast 覆盖层级不一致
- **涉及**：各页面 `uni.showLoading` / `uni.showToast`
- **问题**：部分地方使用 `title` 简略，错误信息截断为40字符可能丢失关键信息
- **建议**：关键错误（如「提交数据为空」）增加详细信息展示

#### 12. 超管「一键重置」操作后未自动退出重新登录
- **文件**：`admin.vue`
- **问题**：重置成功后会刷新数据，但操作日志/状态未清除，可能导致残留
- **建议**：重置成功提示后建议提示「建议重新登录」，或直接跳转登录页

## 三、性能与架构建议

### 3.1 列表分页
目前 `students.vue` 已有前端分页（PAGE_SIZE=20），但 `school-admin.vue`、`admin.vue` 无分页。建议后端统一支持 `?skip=0&take=20` 参数。

### 3.2 图片/文件压缩
`pickFile()` 限制 5MB，`pickImage()` 已有压缩（maxWidth=1280），但学生列表头像等大图未统一压缩。

### 3.3 API 错误处理标准化
当前各页面独立 try-catch + `uni.showToast`，建议统一封装错误处理中间件，在 request.js 中统一处理超时/401/网络异常。
