# 功能细节差异清单（Web ↔ 微信小程序·布局/交互级）

> 目的：在「模块级缺口清单」之后，进一步逐组件对齐 **Web（app/src）** 与 **小程序（mini-program/src）** 的**布局结构、交互细节、功能按钮/字段**，使小程序功能与 Web 尽量一致、布局交互尽量相同。
> 标注说明：`缺失`=Web 有小程序无；`布局不同`=结构/栅格差异；`交互不同`=操作方式差异；`字段不同`=数据模型/字段差异；`小程序多`=小程序反向多出。
> 可行性：`可行`=小程序可实现；`受限`=受小程序平台限制（无 TTS/流式/打印/外链打开等）需替代方案；`概念冲突`=Web 桌面概念与小程序定位不同，需取舍。

### 7.3 统一受限替代方案（沉淀）

- **全屏**：小程序无 `document.fullscreen`，标注「当前平台不支持」。

- **TTS 朗读**：浏览器 `SpeechSynthesis` 不可用，需微信**同声传译插件**（需在 mp 后台「设置→第三方设置→插件管理」添加，且需相应类目资质）；无插件时降级为复制文本。

## 十五、第十轮：MySQL 启动报错修复（`aisleCols` TEXT 列默认值约束）

> 运行时日志报错触发：`[TypeOrmModule] Unable to connect to the database ... QueryFailedError: BLOB, TEXT, GEOMETRY or JSON column 'aisleCols' can't have a default value`。根因为实体定义违反 MySQL DDL 约束，需修正后重新构建推送。

### 15.1 根因与修复

- **现象**：Nest 启动后 TypeORM 同步建表失败，`seat_layouts` 表创建被数据库拒绝，连接反复重试。
- **根因**：`server/src/seats/seat.entity.ts` 中 `@Column('simple-json', { default: '[]' }) aisleCols: number[]`。`simple-json` 在 MySQL 下映射为 TEXT 列，而 MySQL 不允许 `TEXT/BLOB/JSON/GEOMETRY` 类型设置默认值，建表语句 `aisleCols text DEFAULT '[]'` 非法。
- **修复**：改为 `@Column('simple-json', { nullable: true })` —— 去掉默认值、允许为空。`aisleCols`（过道列配置）目前仅实体声明、尚未被 service 读取，置空不影响运行。
- **排查**：已全局扫描 `simple-json|text|json` + `default` 组合，确认无其他同类字段（其余 `simple-json` 列如 `seats` 均无默认值）。
