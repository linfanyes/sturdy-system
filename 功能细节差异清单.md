# 功能细节差异清单（Web ↔ 微信小程序·布局/交互级）

> 目的：在「模块级缺口清单」之后，进一步逐组件对齐 **Web（app/src）** 与 **小程序（mini-program/src）** 的**布局结构、交互细节、功能按钮/字段**，使小程序功能与 Web 尽量一致、布局交互尽量相同。
> 标注说明：`缺失`=Web 有小程序无；`布局不同`=结构/栅格差异；`交互不同`=操作方式差异；`字段不同`=数据模型/字段差异；`小程序多`=小程序反向多出。
> 可行性：`可行`=小程序可实现；`受限`=受小程序平台限制（无 TTS/流式/打印/外链打开等）需替代方案；`概念冲突`=Web 桌面概念与小程序定位不同，需取舍。

### 7.3 统一受限替代方案（沉淀）

- **全屏**：小程序无 `document.fullscreen`，标注「当前平台不支持」。
