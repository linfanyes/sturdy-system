# 功能细节差异清单（Web ↔ 微信小程序·布局/交互级）

> 目的：在「模块级缺口清单」之后，进一步逐组件对齐 **Web（app/src）** 与 **小程序（mini-program/src）** 的**布局结构、交互细节、功能按钮/字段**，使小程序功能与 Web 尽量一致、布局交互尽量相同。
> 标注说明：`缺失`=Web 有小程序无；`布局不同`=结构/栅格差异；`交互不同`=操作方式差异；`字段不同`=数据模型/字段差异；`小程序多`=小程序反向多出。
> 可行性：`可行`=小程序可实现；`受限`=受小程序平台限制（无 TTS/流式/打印/外链打开等）需替代方案；`概念冲突`=Web 桌面概念与小程序定位不同，需取舍。

## 一、日常教学核心（P0/P1）

### 1.1 课表 Schedule
- 课程定位【概念冲突·P0】：Web 是「教师个人课表」只读；小程序是「班级课程表」可编辑。建议保留小程序班级课表，并补齐「任教统计 + 一览 chip + 设置任课弹窗」。
- 任教统计数字 + 任教 chip【缺失·P1·可行】
- 设置任课弹窗（班级×科目勾选 + 清空）【缺失·P1·可行】
- 打印/排版入口【缺失·P2·受限（小程序无打印）】

### 1.2 考勤 Attendance
- 学生字段：性别/头像/座号/学号【字段不同·P0·可行】小程序仅姓名
- 保存方式：Web 点击实时存；小程序需点「保存考勤」批量提交【交互不同·P1·可行（保留批量更稳）】
- 旷课自动公告【缺失·P1·可行】
- 统计：Web 4 图标卡；小程序纯数字条【布局不同·P1·可行】

### 1.3 作业 Homework
- 筛选/排序：Web 只看我的任课 + 学科 + 状态 + 3 种排序【缺失·P0·可行】
- 按班级分组折叠 section【布局不同·P0·可行】
- 开始日期字段【字段不同·P1·可行】
- 已发还自动生成班级公告【缺失·P1·可行】
- 统计项：Web 共/待批改/已完成；小程序 待批改/已处理【字段不同·P2·可行】

### 1.4 公告 Notice
- 范围筛选（全校/各班级 select）【缺失·P0·可行】
- 范围模型不一致：小程序用「全校/本班/年级」虚拟值；Web 用真实 classId 或 '全校'【字段不同·P0·可行/数据互通】
- 两列卡 vs 单列【布局不同·P1·可行】
- 编辑弹窗【缺失·P1·可行】

### 1.5 资源 Resource
- 搜索（标题/标签/描述防抖）【缺失·P0·可行】
- 分类筛选 + 标签 chips【缺失·P0·可行】
- 多列卡 + 分类配色【布局不同·P1·可行】
- 打开链接：Web `<a>` 跳转；小程序复制链接【交互不同·P1·受限（外链打开用 web-view 替代）】

## 四、工具 / 游戏 / AI / 外壳（P0/P1）

### 4.1 工具箱外壳 Toolbox
- Web 7 分类彩色卡 + 「管理分类」可拖改归类持久化；小程序 12 分节门户无分类编辑【布局不同+缺失·P2·可行】

### 4.2 登录 Login
- Web 本地登录码 + 表情头像网格 + 首次设置弹框(学校/学期/学科多选) + 历史账号；小程序仅微信一键登录【交互不同(认证模型)·P0·概念冲突（认证方式不同，保留小程序微信登录）】

### 4.3 导航外壳
- Web 左侧栏分组 + 顶栏(机器人/工具箱/消息铃铛) + 移动底栏 5 格含「消息」+ 今日消息聚合弹框；小程序 tabBar 仅 5 项无消息 tab、无聚合通知【布局不同+缺失·P1·可行(加消息聚合页)】

### 4.4 工具（tools）逐一
- timer：缺秒表/全屏/提示音/预设【功能缺失·P0·可行】
- subject 专项：Web 有 TTS 播报/接龙/评分/朗读；小程序仅文本生成复制【交互不同+缺失·P0·受限(TTS 需插件/录音)】
- decider：缺抛硬币/骰子计数/抽签数量【功能缺失·P1·可行】
- mistakes：缺 7 类分类/编辑弹框/练习卷下载【字段/功能缺失·P1·可行】
- picker：数据源绑班级 store + 排除已点 + 历史；小程序文本录入【数据源/交互不同·P1·可行】
- anscard：Web 固定 200 题+打印+下载；小程序可填答案+批改【交互不同·P1·可行】
- math：缺班级/打印下载【功能缺失·P2·可行】
- vcalc/flower/templates：缺打印下载/通知模板组件【功能缺失·P2·可行】
- calc：缺 `%` 运算符【字段不同·P2·可行】
- unit/multitable：基本对齐【可行】

### 4.5 游戏 games
- 数量对齐（均 18）；差异仅 Web 单网格带标签(数学/经典/对战)，小程序分 2 类(逻辑益智/动作反应)【布局不同·P2·可行】

### 4.6 AI
- Web 一体化多会话+设置+附件+Markdown+流式+智能导入+全量本地上下文；小程序拆 4 页用同步 chat-sync，无会话管理/设置/智能导入/上下文【架构不同+功能缺失·P0·可行(短期补齐附件+复制，会话/流式长期)】

### 4.7 统计 Analysis
- Web 有聚合页(成绩趋势单 Tab)；小程序无独立页，靠 dashboard+ai-exam+grade-trend【布局不同·P1·可行】

## 五、实施建议（按性价比）
1. **先补齐「交互/字段」类且可行的前端差异**（不影响后端，风险低）：Notice 范围筛选+编辑、Resource 搜索+分类+标签、Homework 筛选+分组+排序+开始日期+自动公告、Teacher 增删改、Profile 主题色+数据管理、DutyRoster 多表管理、Attendance 学生字段+旷课公告。
2. **再补「布局」类**：双列卡、统计卡、彩色 chip、时间线。
3. **受平台限制项**（TTS/流式/打印/外链打开/消息聚合）给出替代方案，不强行 1:1。
4. **概念冲突项**（课表定位、登录认证）与用户确认取舍后再动。
