# 园丁工作台 · 微信小程序前后端完整部署与发布流程

> 适用范围：uni-app 小程序（前端）+ NestJS 后端（微信云托管）+ 腾讯云数据库 MySQL 8.0
> 阅读对象：部署/运维负责人
> 本文给出**从代码到生产**的完整流水线：后端上云 → 数据库 → 小程序构建 → 微信公众平台审核发布 → 自检 → 运维。
> 产品使用说明见 **[《系统说明书》](./系统说明书.md)**；开发者视角见 `mini-program/README.md`。

---

| 项目 | 获取方式 | 用途 |

| 小程序 AppID | 微信公众平台 → 开发管理 → 开发设置 | 前后端都绑定它 |

## 二、后端部署（微信云托管）

### 2.1 工程中已就绪的关键文件
- `server/Dockerfile`：多阶段构建，运行阶段以 `node` 用户（非 root）运行。
- `server/cloudbaserc.json`：`cloudRun` 服务声明，端口 `3000`，注入环境变量。
- `server/.env.example`：环境变量模板（含 `DB_SSL` 说明）。
- `server/src/main.ts`：监听 `0.0.0.0:3000`；CORS 已按 `*` 或具体域名条件处理。
- `server/src/app.module.ts`：TypeORM 已适配 MySQL 8.0（`allowPublicKeyRetrieval` + `timezone:+08:00` + `utf8mb4`）。

### 2.2 构建并推送镜像（任选其一）

**方式 A：代码包直传（最简单）**
1. 将 `server/` 目录打包（含 `Dockerfile`、`package.json`、`cloudbaserc.json`）。
2. 云托管 → 新建服务（如 `gardener-server`）→ 部署方式选「代码包」→ 上传 → 自动按 `Dockerfile` 构建。

**方式 B：本地构建镜像推 TCR**
```bash
cd server
docker build -t ccr.ccs.tencentyun.com/<你的命名空间>/gardener-server:1.0 .
docker push ccr.ccs.tencentyun.com/<你的命名空间>/gardener-server:1.0
```
然后在云托管选择该镜像部署。

### 2.3 设置环境变量（云托管控制台）
| 变量 | 示例 | 说明 |
|---|---|---|
| `NODE_ENV` | `production` | 生产环境 |
| `PORT` | `3000` | 容器监听端口 |
| `JWT_SECRET` | 随机长字符串 | JWT 签名密钥，务必复杂且保密 |
| `DB_HOST` | `gz-cdb-xxxx.sql.tencentcdb.com` 或内网 `10.x.x.x` | 数据库地址 |
| `DB_PORT` | `3306` | |
| `DB_USERNAME` | `gardener` | 建议专用最小权限账号，非 root |
| `DB_PASSWORD` | `****` | |
| `DB_DATABASE` | `gardener` | 已存在的库 |
| `DB_SYNCHRONIZE` | `true`（首部署）/ `false`（稳定后） | true 自动建表 |
| `DB_SSL` | `false`（内网）/ `true`（公网） | 公网建议开 SSL |
| `CORS_ORIGIN` | `*` 或 `https://你的域名` | 跨域白名单 |
| `AI_DEFAULT_KEY` / `AI_BASE_URL` | 按需在「设置」页填 | 也可 App 内配置，不入代码 |

> 首部署设 `DB_SYNCHRONIZE=true`，后端启动自动按实体建表；确认表结构正确后改回 `false`，避免被自动覆盖。

### 2.4 部署并拿到访问地址
点击「新建版本并部署」，等待实例就绪（状态绿）。记录服务的**默认域名**（形如 `xxx.ap-shanghai.run.tcloudbase.com`）或你绑定的自定义域名。后续小程序 `API_BASE` 就用它。

## 三、数据库（腾讯云 MySQL 8.0）

### 3.1 创建实例与账号
1. 腾讯云控制台 → 云数据库 MySQL → 新建实例，版本 **MySQL 8.0**，字符集 **utf8mb4**。
2. 新建账号（如 `gardener`），仅授权目标库；记录主机白名单（同 VPC 用内网，跨网临时 `%`）。
3. 新建数据库 `gardener`（utf8mb4 / utf8mb4_general_ci）。

### 3.2 MySQL 8.0 兼容性（已修复，无需改代码）
- **认证**：8.0 默认 `caching_sha2_password`，内网无 TLS 时需 `allowPublicKeyRetrieval=true`（已配）。
- **时区**：`timezone:'+08:00'`，避免时间差 8 小时。
- **字符集**：`charset:'utf8mb4'`。

### 3.3 网络
- 内网：云托管与库同 VPC → 用内网地址（推荐）。
- 公网：数据库开公网，配 `DB_SSL=true`，云托管出口 IP 加白名单。

## 四、小程序构建与发布（微信公众平台）

### 4.1 改 API 基地址
编辑 `mini-program/src/common/config.js`：
```js
export const API_BASE = 'https://你的云托管域名/api'   // 注意带 /api 前缀
```

### 4.2 填 AppID
`mini-program/src/manifest.json` 的 `mp-weixin.appid` 填入你的小程序 AppID（替换占位 `wxYOURAPPID`）。

### 4.3 构建产物
```bash
cd mini-program
npm install
npm run build:mp-weixin      # 产物在 dist/build/mp-weixin
```

### 4.4 配置服务器合法域名（关键，否则真机请求被拦截）
微信公众平台 → 开发管理 → 开发设置 → **服务器域名**：
- `request 合法域名`：`https://你的云托管域名`
- 若用到上传/下载，补充 `uploadFile / downloadFile` 合法域名
> 开发者工具内可临时勾选「不校验合法域名」调试，但**正式版必须配置**。

### 4.5 上传 → 审核 → 发布（发布流水线）
1. **微信开发者工具** → 导入项目 → 目录选 `dist/build/mp-weixin` → 填入 AppID。
2. 工具内「编译」预览，确认页面/功能正常（建议按《小程序冒烟验证清单》走一遍）。
3. 点「**上传**」，填版本号（如 `1.0.0`）与项目备注 → 生成「开发版」。
4. 微信公众平台 → **管理 → 版本管理**：
   - 将「开发版」提交为「**审核版**」（填写功能页面、类目、测试账号等审核信息）。
   - 等待微信审核（通常几小时～1 天）。
   - 审核通过后，在「审核版本」点「**发布**」→ 全量/分阶段发布为「线上版本」。
5. 发布后，用户即可在微信搜索小程序名 / 扫码打开。

> 后续迭代：改代码 → 重新 `build:mp-weixin` → 开发者工具上传新版本 → 审核 → 发布，循环即可。

## 五、上线自检清单

部署+发布后逐项确认（开发者工具 + 真机预览）：

- [ ] **后端健康**：访问 `https://域名/api/health` 或登录接口能连通（无 502/504）。
- [ ] **数据库建表**：首部署 `DB_SYNCHRONIZE=true` 后，数据库出现各业务表（students/exams/grades/seats…）。
- [ ] **小程序能打开**：真机扫码/搜索进入，无白屏、无报错。
- [ ] **登录可用**：`wx.login` → `/api/auth/wechat-login` 返回 JWT，跳工作台。
- [ ] **CRUD 可用**：工具箱各「管理」页增删改查正常（GET/POST/PATCH/DELETE /api/<实体>）。
- [ ] **AI 可用**：「设置」配好密钥后，3 个 AI 生成页 + 语数英专项 + 文字办公能出结果；未配时提示而非崩溃。
- [ ] **CORS/HTTPS**：真机无跨域、全 https（否则被小程序拦截）。
- [ ] **合法域名**：公众平台已登记 `request 合法域名`，正式版请求不失败。
- [ ] **离线工具**：18 款小游戏、随机点名/倒计时/计算器/口算/换算/口诀/错题本/模板库本地可用。

## 六、故障排查

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 小程序请求 `fail` / 404 | API_BASE 未改或域名未备案/未加合法域名 | 改 `config.js` 为正确 https 域名；公众平台加 request 合法域名 |
| 真机打不开/白屏 | AppID 不匹配或域名未配 | 核对 `manifest` AppID；配服务器域名 |
| 后端 `ER_ACCESS_DENIED` / `caching_sha2` | MySQL 8.0 认证 | 确认 `allowPublicKeyRetrieval=true`；公网开 `DB_SSL=true`；账号密码正确 |
| 时间差 8 小时 | 时区 | `timezone:'+08:00'` 已配；确认库 `time_zone` 为 `+08:00` |
| 502 / 504 | 云托管未就绪 / 数据库连接超时 | 看云托管日志；确认 `DB_HOST` 内网可达、白名单放行 |
| CORS 报错 | origin 与 credentials 冲突 | `main.ts` 已按 `*` 或具体域名条件处理；勿同时 `origin:'*'`+`credentials:true` |
| AI 无响应 | 未配密钥 / 模型地址错 | 「设置」页填正确 Key 与 BaseURL；查后端日志 |
| 登录后 401 跳回 | token 失效 / 接口需鉴权 | 确认 JWT_SECRET 一致；`request.js` 自动带 `Authorization: Bearer` |

## 七、运维

- **日志**：云托管控制台「日志」看 stdout；NestJS 全局异常过滤器统一返回结构。
- **扩缩容**：云托管按 CPU/内存弹性伸缩；数据库按需升配。
- **备份**：腾讯云数据库开自动备份；AI 密钥在应用内「设置」持久化，不进仓库。
- **安全**：后端非 root 运行；DB 最小权限账号；JWT_SECRET 随机长串；密钥不入库代码。
- **版本回滚**：云托管与小程序公众平台均支持版本回退，发布异常可快速回滚。
