## 二、前置条件（开始人工冒烟前确认）

- [ ] 后端已部署到云托管且健康检查通过（`GET /api` 或日志无报错）
- [ ] 云数据库 MySQL 8.0 已建库，后端以 `DB_SYNCHRONIZE=true` 完成首次建表
- [ ] 小程序 `config.js` 的 `API_BASE` 已指向后端可达域名（开发者工具内因 `urlCheck:false` 任意域名均可；真机发布需在小程序后台登记 request 合法域名）
- [ ] `manifest.json` 已填真实 AppID；开发者工具已「导入项目」选中 `dist/build/mp-weixin`
- [ ] 后端「设置」已配置 AI 密钥（仅 AI 类工具需要）

## 三、启动与登录

- [ ] 开发者工具打开项目，编译无红色报错；模拟器显示「园丁工作台」登录页（🌻 图标）
- [ ] 点「微信登录」→ 调起 `uni.login()` 拿到 code → 请求 `/api/auth/wechat-login`
  - 首次登录：提示「欢迎！请先完善资料」并跳「设置」页（switchTab 到 config）
  - 老用户：直接进「工作台」
- [ ] 杀进程重进：因 token 存 `uni.storage`，应自动跳过登录（若 `onShow` 检测无 token 才回登录页）
- [ ] 模拟 401：后端返回 401 时任一页应自动 `reLaunch` 到登录页

## 四、核心数据流程（CRUD 类）

逐项点开工具箱 → 对应 crud 页，验证「列表加载 / 新增 / 编辑 / 删除 / 搜索」：

- [ ] 班级管理 / 学生管理：新增学生→列表出现→编辑→删除生效
- [ ] 座位表：分配学生、保存、激活布局（回写学生行列）
- [ ] 随机分组：按组数/每组人数生成；按列分组读座位布局
- [ ] 考试管理：新建考试（默认勾选语数英科+科学+品德）→ 自动建成绩记录
- [ ] 成绩管理：导入/录入成绩→排名与统计显示
- [ ] 课表/考勤/作业/公告/资源/教案/听课记录：增删改查正常
- [ ] 班级事务（轮值/班费/活动/待办/笔记/抽签历史）：正常
- [ ] 学生发展（成长/行为/获奖/加减分/积分/小组评分/家长联系）：正常
- [ ] 教师行政（通讯录/工作日志）：正常

> 判定：每个实体至少跑通「新增→列表可见→编辑→删除」闭环，且刷新后数据仍在（持久化到后端 MySQL）。
