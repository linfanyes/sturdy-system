# 部署就绪检查清单

> 针对「园丁工作台」微信小程序生产部署的逐项检查清单
> 适用对象：后端 NestJS 服务 + 微信小程序前端

---

## 🔴 优先级 P0 — 不改不能上线

### 1. 密钥与凭据
| 检查项 | 当前值 | 生产要求 | 操作 |
|--------|--------|---------|------|
| `JWT_SECRET` | `test_jwt_secret_…` | 32+ 字节随机字符串 | 生成新密钥 |
| `SUPER_ADMIN_PASSWORD` | `admin` | 12+ 位强口令 | 修改 |
| `WECHAT_APPID` | `wx_test_appid` | 真实微信小程序 AppID | 替换 |
| `WECHAT_SECRET` | `test_wechat_secret` | 真实微信小程序 AppSecret | 替换（**切勿提交到代码仓库**） |
| `AI_API_KEY` | `空` 或测试 key | 真实 API Key | 填写 |
| `DB_PASSWORD` | `admin` | 强密码 | 修改 |

### 2. 网络安全
| 检查项 | 当前值 | 生产要求 | 备注 |
|--------|--------|---------|------|
| `CORS_ORIGIN` | `*` | 限定为小程序域名/Web 域名 | 如 `https://yourdomain.com` |
| 小程序域名白名单 | 未配置 | 在微信公众平台配置 `request` / `uploadFile` / `downloadFile` 合法域名 | 务必部署至 HTTPS |
| 后端 HTTPS | ❌ 未配置 | Nginx/Caddy 反向代理 + Let's Encrypt | 微信小程序要求所有请求 HTTPS |
| `DB_HOST` | `127.0.0.1` | 内网数据库 IP（不应暴露公网） | 确认服务器间内网可达 |
| `DB_SSL` | `false` | `true` | 生产数据库开启 SSL 连接 |

### 3. 微信小程序审核
| 检查项 | 要求 | 当前状态 |
|--------|------|---------|
| 小程序类目 | 选择教育类（「教育」→「学校」） | 需在微信公众平台提交 |
| 隐私协议 | `app.json` `__usePrivacy__` 声明 + 弹窗授权 | 需检查是否实现 |
| 用户授权弹窗 | 获取用户信息/位置/相册前必须弹窗征得同意 | 需逐页检查 |
| 虚拟支付 | 教育类小程序禁止未备案虚拟支付（iOS 端） | 如涉及需确认合规 |
| 内容安全 | 用户上传文本/图片需调用 `msgSecCheck` / `imgSecCheck` | ✅ 已实现 `/security/msg-check` |
| 用户身份 | 必须使用 `wx.login` 获取 code 换取 session_key | ✅ 已实现 `/auth/wechat-login` |
| 小程序名称 | 与功能介绍、服务范围一致 | 确认 |

## 新增功能的上线注意事项

### 超管配置页（`pages/admin/admin.vue`「配置」Tab）
- 依赖微信云托管 `wx.cloud.callContainer`，本地开发不可用
- 密钥类字段（apiKey、AppSecret）前端脱敏显示，写入时仍接收明文
- 修改 AI/微信/IM 配置后**即时生效**，无需重启服务

### 校管班级管理（`/school-admin/classes` 端点）
- 创建班级时必须选择班主任（教师必须本校）
- 删除班级**不会迁移**关联学生数据，需教师端手动处理

### 校管学生管理（`/school-admin/students` 端点）
- 只读列表（展示学生姓名/学号/班级/家长开通状态）
- 不包含学生编辑/删除功能，班级内学生管理仍需教师端操作

### 学校公告（`/school-admin/notices` 端点）
- 校管创建公告 → `scope='school'` → 教师 Dashboard 自动显示
- 教师端首页使用 `GET /notices?scope=school` 拉取
- 校管可删除自己发布的公告

### 家长中心 Tab 改造
- 「待办公告」Tab：作业列表 + 班级公告，空态友好提示
- 「成绩查询」Tab：原考试全部功能（选考试→看总分/排名/科目/优劣势/分布图/分析）

### 教师 Dashboard 增强
- 快捷操作栏：记考勤 / 布置作业 / 发通知 / 待办 / 行为记录
- 学校公告区：校管发布的公告带红色未读计数徽标
- 学校公告通过 `GET /notices?scope=school` 获取

### 微信推送集成
- 教师发公告时可勾选「推送给家长」
- 后端自动查出该班级已绑微信的家长 openId，调用微信订阅消息 API
- 家长端顶部绿色引导卡片引导订阅
- 模板 ID 替换（`security.module.ts`）后自行生效
- 未配置 AppSecret 时推送静默跳过

### 角色守卫体系
| 角色 | API 范围 | 实现 |
|------|---------|------|
| 超管 | `admin/*` | `SuperAdminGuard` |
| 校管 | `school-admin/*` | `SchoolAdminGuard` |
| 教师 | 33 个业务 CRUD 端点 | `@Roles('teacher')` + `JwtAuthGuard` |
| 家长 | `parent-auth/*` | `ParentAuthGuard` |
| 跨角色 | `security/*` | `@Roles('teacher', 'parent')` |

## 新环境首次启动流程

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env → 填入真实值（JWT_SECRET/DB_PASSWORD/WECHAT_* 等）

# 2. 编译后端
cd server
npm install
npx nest build

# 3. 启动（种子数据自动重建）
node dist/main.js

# 4. 验证自动种子
curl http://localhost:3000/api/auth/unified-login \
  -H "Content-Type: application/json" \
  -d '{"username":"teacher1","password":"123456"}'
# → 返回 token 和用户信息即成功

# 5. 运行 E2E 验证
NODE_PATH=server/node_modules node test-deliverables/e2e-journey.js
```
