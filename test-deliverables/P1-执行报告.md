## 三、发现（后端 ↔ 前端词汇不一致，建议后续修复）

### ① 考勤状态枚举：真实 API 返回英文，前端仅识别中文
- 真实 `attendances.records[].status` 返回 `present / late / absent`（live API 实测）。
- 前端 `attendance.vue` 全程使用中文（`出勤/迟到/请假/旷课`），`stats[status]++` 与 `statusMap[status]` 均为中文键。
- **影响**：真实模式下统计数据会全部归零（键不匹配），演示模式因 mock 用中文而正常。
- **建议**：后端在返回前做 `present→出勤` 映射，或前端在 `loadAttendance` 处加归一化。本次 P1-B 保持 mock 用中文以确保演示可用，未改动后端。

### ② 值日表 type 字段：真实 API 返回 `weekly`，前端期望 `值日/班干部`
- 真实 `duty_rosters.type` 存 `'weekly'/'daily'`（频率语义）。
- 前端 `duty-roster.vue` 的标签逻辑 `r.type === '值日' ? 't-zh' : 't-bn'`，选项为 `值日/班干部`（职责语义）。
- **影响**：真实模式标签文案显示 `weekly` 且套用班干部样式；演示模式已对齐为 `值日`。
- **建议**：统一 `type` 的语义（建议前端改为频率枚举，或后端改存职责枚举），保持前后端一致。
