## 👑 超级管理员（超管）

### 当前功能
| 模块 | 现有能力 |
|------|---------|
| 学校管理 | ✅ CRUD、启用/停用 |
| 学校管理员管理 | ✅ CRUD、密码重置 |
| 数据重置 | ✅ `reset-all`（已加 `confirm` 验证） |

### 改进建议

#### P1（高价值/低工作量）

**1. 平台配置管理页**
- 超管当前只能在 `.env` 中改配置，小程序端没有可视化界面
- 后端已有 `PUT /config/app/:key` 接口（超管权限），只需要在 `admin.vue` 中加一个 Tab 页
- 可配置项：默认学科、AI 模型/温度、微信 AppID、IM 参数
- **改动量**：前端 ~200 行，后端不需改

**2. 各学校使用统计**
- 当前超管只能看到学校列表，看不到运营数据
- 后端新增 `GET /admin/school-stats` 接口 ← 刚才确认没有此接口
- 返回每所学校的：教师数、班级数、学生数、活跃度
- **改动量**：后端 ~30 行，前端 ~100 行

**3. 重置确认参数对齐**
- `admin.vue` 调用 `reset-all` 未传 `{ confirm: true }`，后端已要求确认
- **改动量**：前端 1 行（追加参数）

#### P2（中价值/中等工作量）

**4. 操作审计日志**
- 记录学校管理员的关键操作（新增/删除教师、重置密码）
- 新增 `audit_logs` 表 + 超管查看界面
- **改动量**：后端 1 表 + 1 接口 ~80 行，前端 1 页 ~150 行

**5. 数据导出**
- 学校/教师列表可导出 CSV/Excel
- 后端 `json2csv` 转换，或复用已有 excel.ts 工具
- **改动量**：后端 ~40 行，前端 ~50 行

## 🏫 学校管理员（校管）

### 当前功能
| 模块 | 现有能力 |
|------|---------|
| Dashboard | ✅ 教师/班级/学生概览 |
| 教师管理 | ✅ CRUD、功能配置、密码重置 |
| 家长登录管理 | ✅ 列表查看 |

### 改进建议

#### P1（高价值/低工作量）

**1. 班级管理**
- 校管可以新增/编辑/删除班级，指定班主任（授课教师）
- 后端已有 `classes` 表、CRUD 接口（目前仅教师可操作），只需给校管也开放接口
- 新增 `PATCH /school-admin/classes/:id`、`POST /school-admin/classes`
- **改动量**：后端 ~50 行，前端 ~200 行（复用班级管理表单）

**2. 学校公告**
- 校管可以发一条置顶公告，所有教师登录后看到
- 新增 `notices` 表复用现有结构（增加 `scope: 'school'` 字段区分校管公告 vs 班级公告）
- **改动量**：后端 ~30 行，前端 ~120 行

**3. Dashboard 丰富**
- 当前只看数字概览，缺少可视化
- 增加今日活跃教师数、待办总数、近七日考勤率趋势
- **改动量**：后端 ~50 行（新增聚合接口），前端 ~150 行（图表卡片）

#### P2（中价值/中等工作量）

**4. 学期管理**
- 校管可以创建/关闭学期
- 后端已有 `semesters` 表，校管可操作
- **改动量**：后端 ~40 行，前端 ~100 行

**5. 教师课表总览**
- 查看全校教师的课表总览，方便排课协调
- **改动量**：后端 ~30 行，前端 ~150 行

**6. 学生批量操作**
- 批量导入学生（Excel/CSV）、批量分班、批量重置家长登录
- **改动量**：后端 ~80 行，前端 ~200 行
