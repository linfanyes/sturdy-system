## 3. 缺陷清单

> 状态：已修复并验证 = 代码已改，且经源码走查 + 编译 +（D1）运行复现三重验证；阻塞 = 需运行环境无法自动验证（功能类）。

| 缺陷编号 | 所属模块 | 严重等级 | 复现步骤 | 当前状态 |
|---|---|---|---|---|
| D1 | backup | 严重 | 教师 A 登录后 `GET /api/backups`；控制器用 `req.user.teacherId||req.user.id`（恒 undefined）作 teacherId，TypeORM 丢弃 WHERE → 返回/可删全部教师备份 | 已修复并验证 |
| H1 | config | 严重 | 普通教师 `GET /api/config/app` 仅过 JwtAuthGuard，返回含 wxAppSecret/imSecretKey/aiApiKey/loginCode 的明文；`PUT` 可覆写 | 已修复并验证 |
| S2 | 启动/JWT | 严重 | `JWT_SECRET` 为默认占位值且未强制；攻击者可伪造任意 JWT 越过鉴权 | 已修复并验证 |
| S1 | auth | 严重 | 未设环境变量时超管默认 admin/admin；`.env.example` 原缺该项 | 已修复并验证 |
| H2 | grades | 高 | `GET /api/grades`（不带 classId）返回全部教师成绩；带他人 classId 泄露其成绩 | 已修复并验证 |
| H3 | exams | 高 | 同 H2，考试实体跨租户泄露 | 已修复并验证 |
| H4 | school/notices | 高 | `POST /api/notices/push` 按 classId 查学生并回显姓名+openId，无班级归属校验 | 已修复并验证 |
| H5 | ai | 高 | `POST /api/ai/analyze-exam` 按 examId 查考试无 teacherId 校验，可分析他人考试 | 已修复并验证 |
| H6 | ai | 中高 | `POST /api/ai/diagnose` 按 studentId 查学生无 teacherId 校验 | 已修复并验证 |
| M1 | im | 中 | `POST /api/im/user-sig` 接受客户端 userId，可为任意 IM 身份签发 UserSig | 已修复并验证 |
| M2 | 通用 CRUD | 中 | 写操作 body 含 `teacherId`/`role` 等字段未被剥离（teacherId 仅 create 覆盖） | 已修复并验证 |
| S3 | wechat | 中 | `wechat.service.ts` 全局 `rejectUnauthorized:false`，存在 MITM 窃取 openid/session_key 风险 | 已修复并验证 |
| A | App.vue | 高(前端) | 冷启动仅判断 `auth.token`（教师令牌），超管/校管/家长令牌存在也被退回登录页 | 已修复并验证 |
| B | login.vue | 高(前端) | 超管/校管 token 仅存 `admin_token`/`sa_token`，共享 api 层读 `g_token` 为空 → Bearer 缺失 401 | 已修复并验证 |
| C | request.js | 中(前端) | `setMockMode` 不写 `g_mock_mode`；`App.vue` 读该键但永远读不到 → 演示模式重启丢失 | 已修复并验证 |
| D | request.js/store | 中(前端) | 401 调教师 `logout()` 仅清 `g_token`，家长令牌残留；且统一跳教师登录 | 已修复并验证 |
| E | request.js | 低(前端) | `REQUEST_TIMEOUT=30000` 但超时文案写「15s」 | 已修复并验证 |

## 5. 结论与建议
- 核心风险集中在**租户隔离被手写覆盖绕过**与**平台密钥接口缺超管保护**，已全部修复。
- 建议后续：补充自动化集成测试（接入 MySQL 实例）、对 `config/app` 之外的敏感接口做一轮安全走查、将前端多角色会话统一为单一令牌模型以减少维护面。
