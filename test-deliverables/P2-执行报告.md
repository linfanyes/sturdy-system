## 一、两项已发现的词汇不一致修复

### ① 考勤状态枚举：后端英文 → 前端中文

**问题**：`attendances.records[].status` 种子数据使用了英文（`present/late/absent/leave`），但前端 `attendance.vue` 全程使用中文（`出勤/迟到/请假/旷课`），真实模式统计会归零。

**修复**：
- 前端 `attendance.vue` 的 `loadAttendance()` 函数增加 `STATUS_NORMALIZE` 映射（present→出勤、late→迟到、absent→旷课、leave→请假），防御归一化避免未来再有数据混入英文值。
- DB 执行 `UPDATE attendances SET records = REPLACE(...)` 将现有 3 条考勤记录的英文状态全量转为中文。
- 种子脚本 `d-seed-core-tables.js` 中 attendances 部分（L132/L139-142）将 `'present'/'late'/'absent'/'leave'` 改为 `'出勤'/'迟到'/'旷课'/'请假'`，永久根治。

### ② 值日表 type 枚举：后端 `weekly` → 前端 `值日`

**问题**：`duty_rosters.type` 种子数据使用 `'weekly'`，前端 `duty-roster.vue` 的标签逻辑为 `'值日'/'班干部'`，导致标签文案不匹配。

**修复**：
- DB 执行 `UPDATE duty_rosters SET type = '值日' WHERE type != '值日'`（2 行）。
- 种子脚本 `d-seed-core-tables.js` 中 duty_rosters 部分（L328/L331）将 `'weekly'` 改为 `'值日'`。

## 二、P2：索引覆盖优化

### 分析过程
1. 通过 `EXPLAIN FORMAT=JSON` 对 `CrudService.findAll()` 的典型查询（`WHERE teacherId=? ORDER BY createdAt DESC`）逐表分析。
2. 确认现有 `idx_teacher / idx_teacher_class` 索引覆盖了 WHERE 过滤，但 `ORDER BY createdAt DESC` 导致 **filesort**。
3. `COUNT(*)` 查询已完全覆盖（仅用索引，扫描 3 行）。
4. 当前数据集很小（最大表 16 行），filesort 无实际性能影响；但为应付增长，提前添加覆盖索引。
