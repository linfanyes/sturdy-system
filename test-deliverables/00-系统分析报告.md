# 园丁工作台 · 微信小程序 · 系统分析报告

> 分析对象：微信小程序端（uni-app → mp-weixin）+ 配套 NestJS 后端 + 腾讯云数据库 MySQL 8.0
> 分析范围：前端页面交互、后端接口逻辑、数据流转、权限控制
> 分析方式：源码走查（server/src、mini-program/src）+ 关键缺陷可执行复现

---

## 1. 系统概述

园丁工作台是面向班主任/学科教师的班级管理工具，小程序端把桌面端核心能力搬到微信内。整体调用链：

```
微信小程序(uni-app) ──HTTPS/云托管私有链路──► NestJS 后端 ──► MySQL 8.0
       登录态: JWT(Bearer)          鉴权: JwtAuthGuard          数据: 按 teacherId 租户隔离
```

- **实体规模**：28 个业务实体全部继承 `BaseEntity`（含 `teacherId` 租户键）；~40 条后端路由。
- **角色**：`teacher`（教师，主角色）、`super`（超管）、`school_admin`（校管）、`parent`（家长）。
- **隔离机制**：`BaseEntity.teacherId` + `CrudService`（写操作自动注入 `teacherId`）+ `JwtAuthGuard`（解析 JWT → `req.user={sub,openid}`）。

## 2. 前端页面交互分析（mini-program/src）

| 维度 | 实现 | 结论 |
|---|---|---|
| 请求封装 | `common/request.js` 走 `wx.cloud.callContainer` 私有链路，自动带 `Bearer`；`put→PUT`、`patch→PATCH` 映射正确 | 基本正确 |
| 登录链路 | `pages/login/login.vue`：`uni.login()`/统一登录 → 后端 → 按 `role` 分流 | 多角色 token 存储不一致（见缺陷 B） |
| 启动网关 | `App.vue onLaunch`：仅判断 `auth.token`（教师令牌） | 超管/校管/家长冷启动被误判未登录（缺陷 A） |
| 演示模式 | `setMockMode` 切本地 mock 数据；`App.vue` 读取 `g_mock_mode` | 开关从未被持久化（缺陷 C） |
| 401 处理 | `request.js` 401 分支调用教师 `logout()` 并 `reLaunch` 到教师登录 | 角色无关，家长/管理员登出不彻底（缺陷 D） |
| 超时提示 | `REQUEST_TIMEOUT=30000` 但文案写「15s」 | 文案与常量不一致（缺陷 E） |

> 详见证缺陷清单 D1–D5。

## 3. 后端接口逻辑分析（server/src）

### 3.1 认证与授权
- `auth/auth.service.ts`：`wechatLogin` 签发 `{ sub: userId }`（无 role）；`unifiedLogin` 中超管签发 `{ sub:'super', role:'super' }`。
- `common/guards/jwt-auth.guard.ts`：解析 JWT → `req.user={ sub, openid }`，**不校验 role**。
- `admin/super-admin.guard.ts`：校验 `p.role==='super'`。

### 3.2 通用 CRUD
- `common/crud/base.service.ts`：`findAll/One/create/update/remove` 全部带 `teacherId` 过滤；`create` 用 `t.sub` 覆盖 `teacherId`（防伪造）。**基类隔离正确**。
- 风险来自**手写覆盖/独立控制器**绕过基类（见缺陷）。

### 3.3 配置模块（高危）
- `config/config.controller.ts`：`GET/PUT /api/config/app` 仅 `JwtAuthGuard`，`listAppConfig()` 返回**全部** `app_config` 行，含 `wxAppSecret`、`imSecretKey`、`aiApiKey`、`loginCode` 等密钥 → 任意登录用户可读密钥并可覆写（缺陷 H1）。

### 3.4 AI / IM / 学校通知
- `ai/ai.controller.ts`：`analyze-exam` 按 `examId` 查考试**无 teacherId 校验**；`diagnose` 按 `studentId` 查学生**无 teacherId 校验** → 跨租户读取（缺陷 H5/H6）。
- `im/im.module.ts`：`user-sig` 接受客户端 `userId` → 可为任意 IM 身份签发 UserSig（缺陷 M1）。
- `school/school.module.ts`：`notices/push` 按 `classId` 查学生并**回显家长 openId/姓名**，无班级归属校验（缺陷 H4）。

## 4. 数据流转分析

### 4.1 正常链路
```
小程序 api.get('/students')
  → request.js 注入 Authorization: Bearer <g_token>
  → 后端 JwtAuthGuard 解析 → req.user.sub = teacherId
  → CrudController → CrudService.findAll(teacherId) → WHERE teacherId = ?
  → MySQL 返回该教师数据
```

### 4.2 演示模式链路
```
setMockMode(true) → _mockMode=true → api.* 直接返回 mock-data.js 本地数据
（不落库，重启后由 g_mock_mode 恢复 —— 但此前未持久化）
```

### 4.3 隔离失效链路（缺陷本质）
所有隔离失效均因**手写查询未带 `teacherId` 或未校验资源归属**：
- 控制器用 `req.user.teacherId || req.user.id`（恒为 undefined）→ WHERE 被丢弃（backup）。
- 覆盖 `findAll` 仅按 `classId` 过滤且未校验班级归属（grades/exams）。
- 独立接口按 `id`/`classId` 直查未校验 owner（ai/school）。

## 5. 权限控制分析

| 角色 | Token 载荷 | 可访问端点 | 风险 |
|---|---|---|---|
| teacher | `{sub}` | 各 CRUD + ai/im/school（JwtAuthGuard） | 跨租户读他人数据（H2/H3/H4/H5/H6） |
| super | `{sub:'super', role:'super'}` | 需 SuperAdminGuard（admin/*、现新增 config/app） | 默认弱口令 admin/admin（S1） |
| school_admin | `{sub, role:'school_admin', schoolId}` | school-admin/*、部分 CRUD | 同上 |
| parent | `{sub, type:'parent', studentId}` | parent-auth/* | 低 |

**权限缺陷汇总：**
1. 租户隔离在多处被绕过（H2/H3/H4/H5/H6、D1）。
2. 平台密钥接口无超管保护（H1）。
3. 默认超管弱口令 + JWT 默认密钥（S1/S2）。
4. 越权批量赋值（mass assignment，M2）。
5. 前端多角色会话/登出处理不一致（D1–D5）。

## 6. 已识别风险点（待测试验证 → 见缺陷清单）

| 编号 | 模块 | 类型 | 严重等级 |
|---|---|---|---|
| D1 | backup | 跨租户越权(读/删全部备份) | 严重 |
| H1 | config | 密钥泄露+可被覆写 | 严重 |
| H2 | grades | 成绩跨租户泄露 | 高 |
| H3 | exams | 考试跨租户泄露 | 高 |
| H4 | school(notices/push) | 家长 PII 泄露 | 高 |
| H5 | ai(analyze-exam) | 考试数据跨租户 | 高 |
| H6 | ai(diagnose) | 学生数据跨租户 | 中高 |
| M1 | im(user-sig) | IM 身份伪造 | 中 |
| M2 | 通用 CRUD | 越权批量赋值 | 中 |
| S1 | auth | 默认超管 admin/admin | 安全 |
| S2 | 启动 | JWT_SECRET 默认占位值可伪造 | 安全 |
| S3 | wechat | TLS 校验被关闭(MITM) | 安全 |
| A | App.vue | 多角色冷启动登出 | 高(前端) |
| B | login.vue | 超管/校管 token 未注入共享 api | 高(前端) |
| C | request.js | 演示模式不持久化 | 中(前端) |
| D | request.js/logout | 401 角色无关登出 | 中(前端) |
| E | request.js | 超时文案与常量不符 | 低(前端) |
