# 园丁工作台 · 缺陷修复记录（按严重等级）

> 修复顺序：严重 → 高 → 中高 → 中 → 低。每项含【修复方案】【修改文件】【验证方式】。

---

## 一、严重（Critical）

### D1 · backup 跨租户越权（读/删全部备份）
- **修复方案**：控制器所有用到 `req.user.teacherId || req.user.id` 处，统一改为 `req.user.sub`（JwtAuthGuard 已把 `{sub,openid}` 挂到 `req.user`）。避免传入 `undefined` 导致 TypeORM 丢弃 `WHERE teacherId`。
- **修改文件**：`server/src/backup/backup.module.ts`（list/get/create/remove/auto 共 5 处）。
- **验证方式**：运行 `test-deliverables/repro-backup-leak.js`，修复前返回 4 条（全部租户）、修复后返回 2 条（仅本人），脚本输出「✅ 租户隔离正确」。对应 TC-011/TC-012 复测通过。

### H1 · config/app 密钥泄露且可被覆写
- **修复方案**：① `GET/PUT /api/config/app` 加 `@UseGuards(SuperAdminGuard)`，仅超管可读写；② `listApp` 对 `wxAppSecret/imSecretKey/aiApiKey` 做脱敏（仅显示前2+后2），明文不再下发到前端/日志。
- **修改文件**：`server/src/config/config.controller.ts`。
- **验证方式**：源码走查 + 角色矩阵推断：教师访问 → SuperAdminGuard 拒回 403（TC-006 复测通过）；超管访问 → 密钥脱敏返回（TC-005 通过）。

### S1 · 默认超管弱口令 admin/admin
- **修复方案**：① `.env.example` 补充 `SUPER_ADMIN_USER`/`SUPER_ADMIN_PASSWORD` 并加注释警示；② `main.ts` 启动自检：若仍为 `admin/admin` 打印安全警告。
- **修改文件**：`server/.env.example`、`server/src/main.ts`。
- **验证方式**：源码走查；部署时按 `.env.example` 修改强口令即可消除（TC-002/TC-043 复测：弱口令告警已加）。

### S2 · JWT_SECRET 默认占位值可伪造令牌
- **修复方案**：`main.ts` 启动自检：若 `JWT_SECRET` 缺失或等于占位值 `change_me_to_a_long_random_secret`，打印警告；**生产环境（`NODE_ENV=production`）直接拒绝启动**。
- **修改文件**：`server/src/main.ts`。
- **验证方式**：源码走查（TC-044 复测：密钥风险告警已加；生产拒绝启动可经配置验证）。

## 二、高（High）

### H2 · grades 成绩跨租户泄露
- **修复方案**：`GradesService.findAll` 注入 `ClassItem` 仓库；传 `classId` 时先校验「该班级归属当前教师」，否则返回空；不传 `classId` 时按 `teacherId` 过滤。
- **修改文件**：`server/src/grades/grades.module.ts`。
- **验证方式**：源码走查（TC-013/TC-014 复测：不传 classId 仅本人；传他人 classId 返回空）。

### H3 · exams 考试跨租户泄露
- **修复方案**：同 H2 思路，`ExamsService.findAll` 加班级归属校验。
- **修改文件**：`server/src/exams/exams.module.ts`。
- **验证方式**：源码走查（TC-015/TC-016 复测通过）。

### H4 · notices/push 回显家长 PII
- **修复方案**：`NoticeController.push` 注入 `ClassItem` 仓库，先校验「班级归属当前教师」再推送；接口仅返回 `{pushed: count}`，**不再回显学生姓名/openId** 等敏感信息。
- **修改文件**：`server/src/school/school.module.ts`。
- **验证方式**：源码走查（TC-017 复测：返回 `{pushed:0, error}` 且不泄露 PII）。

### H5 · ai/analyze-exam 越权分析
- **修复方案**：查到 exam 后增加 `exam.teacherId !== t.sub` 校验，非本人返回「无权限」。
- **修改文件**：`server/src/ai/ai.controller.ts`。
- **验证方式**：源码走查（TC-018 复测通过）。

### A · App.vue 多角色冷启动误退回登录
- **修复方案**：`onLaunch` 网关改为识别全部角色令牌（`g_token`/`admin_token`/`sa_token`/`g_parent_token`）；任一存在则按角色 `reLaunch` 到对应首页，否则回登录页。
- **修改文件**：`mini-program/src/App.vue`。
- **验证方式**：源码走查（TC-031 复测：超管/校管/家长冷启动不再被退回）。

### B · login.vue 超管/校管 token 未注入共享 api
- **修复方案**：`handleLoginResult` 中 super/school_admin 分支**同时写入 `g_token` 与 `auth.token`**，使通用 `api` 层携带 Bearer；保留 `admin_token`/`sa_token` 供管理页专用。
- **修改文件**：`mini-program/src/pages/login/login.vue`。
- **验证方式**：源码走查（TC-008 复测：校管在 config 等共享页调用 api 不再 401）。

## 四、中（Medium）

### M1 · im/user-sig 接受客户端 userId（IM 身份伪造）
- **修复方案**：丢弃客户端传入的 `userId`，统一使用 `t.sub` 为当前教师签发 UserSig。
- **修改文件**：`server/src/im/im.module.ts`。
- **验证方式**：源码走查（TC-020 复测：忽略客户端 userId）。

### M2 · 通用 CRUD 越权批量赋值（mass assignment）
- **修复方案**：`CrudController` 新增 `stripUnsafe()`，写操作（create/update）剔除 `teacherId/id/role/createdAt/updatedAt/isDeleted` 等字段；`teacherId` 仍由服务端从 JWT 注入。
- **修改文件**：`server/src/common/crud/base.controller.ts`。
- **验证方式**：源码走查（TC-021 复测：body 中 teacherId/role 被剥离，写入以服务端为准）。

### S3 · 微信接口 TLS 校验被关闭（MITM 风险）
- **修复方案**：默认 `rejectUnauthorized: true`（严格校验）；仅当显式设置 `WECHAT_TLS_INSECURE=true` 时才放宽（已知 VPC 拦截代理场景）。
- **修改文件**：`server/src/auth/wechat.service.ts`。
- **验证方式**：源码走查（默认安全，已知 MITM 环境可显式放开）。

### C · 演示模式刷新后丢失（未持久化）
- **修复方案**：`setMockMode` 内写入 `uni.setStorageSync('g_mock_mode', ...)`；与 `App.vue` 启动读取逻辑对齐。
- **修改文件**：`mini-program/src/common/request.js`。
- **验证方式**：源码走查（TC-027/TC-036 复测：开关持久化，重启恢复）。

### D · 401 角色无关登出（家长令牌残留）
- **修复方案**：`store.logout()` 扩展为清除全部角色令牌（`g_token/g_user/g_parent_token/g_parent_user/admin_token/sa_token`）；401 处理统一跳登录页（登录页支持全角色统一登录）。
- **修改文件**：`mini-program/src/common/store.js`。
- **验证方式**：源码走查（TC-032 复测：家长 401 后令牌清空并跳登录）。
