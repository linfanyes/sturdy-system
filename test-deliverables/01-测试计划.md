# 园丁工作台 · 微信小程序 · 测试计划

## 1. 测试范围

### 1.1 覆盖模块
| 类别 | 模块 |
|---|---|
| 认证与权限 | 微信登录、统一登录、JWT 鉴权、超管/校管/家长多角色、租户隔离 |
| 核心业务 | 班级、学生、考试、成绩、课表、考勤、作业、公告、资源 |
| 配置与安全 | 平台配置(app)、AI 配置、密钥下发、默认超管 |
| AI 与通讯 | 考试分析、学情诊断、IM UserSig、学校通知推送 |
| 数据管理 | 备份与恢复、演示模式、通用 CRUD |
| 前端交互 | 启动网关、登录分流、401 处理、演示模式持久化、请求封装 |

### 1.2 不覆盖
- 第三方支付（本项目无支付模块）。
- 真机手势/振动/剪贴板等硬件能力（仅做静态走查，标注 Blocked）。
- 性能/压力/弱网（超出本次范围，见冒烟清单）。

## 2. 测试环境

### 2.1 默认配置（按用户要求，使用默认配置）
- 后端：NestJS + MySQL 8.0（开发态 `DB_SYNCHRONIZE=true`）。
- **默认超管账户**：`admin / admin`（环境变量 `SUPER_ADMIN_USER`/`SUPER_ADMIN_PASSWORD` 未配置时生效）。
- JWT：`JWT_SECRET` 取默认占位值（测试环境允许）。
- 前端：uni-app 编译为 mp-weixin；`API_BASE` 指向本地/云托管可达地址；演示模式默认关闭。

### 2.2 测试账号（构造）
| 角色 | 账号 | 说明 |
|---|---|---|
| 超管 | admin / admin | 默认超管 |
| 教师 A | teacherA（由校管创建，密码自定） | 拥有班级 c-A |
| 教师 B | teacherB | 拥有班级 c-B（与 A 无交集） |
| 家长 | 学号 + 默认密码 123456 | 绑定学生 s1 |

### 2.3 执行约束说明
- **后端接口测试**：因本环境未提供 MySQL 实例，无法启动完整 Nest 服务；采用「源码逻辑走查 + 关键缺陷可执行复现脚本（Node）」方式验证，结论以源码为准并标注。
- **前端功能测试**：需微信开发者工具/真机，本环境无法运行；采用「源码静态走查」验证，真机/模拟器相关用例标注 Blocked。

## 3. 测试策略

### 3.1 功能测试（Functional）
验证各模块「新增→列表可见→编辑→删除」闭环、登录分流、演示模式、配置读写等正常路径。

### 3.2 接口测试（API / Integration）
验证请求封装（Bearer、方法映射、401）、后端路由鉴权、租户隔离、密钥接口保护。

### 3.3 边界测试（Boundary）
- 空数据 / 空列表返回。
- 不存在的资源 id（返回 404/空）。
- 分页 `skip/take` 边界（0、超大值）。
- 越权 `classId`（传他人班级 id）。
- 超长字段 / 特殊字符。

### 3.4 异常测试（Exception）
- 无 token / 失效 token → 401。
- AI 未配置密钥 → 友好提示不崩溃。
- 网络异常 → 超时与错误归一化。
- 并发绑定（已知有悲观锁，验证不重复绑定）。
- 默认超管弱口令（安全异常）。
