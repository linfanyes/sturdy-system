# 园丁工作台 · 当前项目 vs 原始 Web 项目 功能缺口对比清单

> 生成时间：2026-07-18
> 对比对象：
> - **原始 Web 项目** `work-system/app`（Vue3 + Vite + Pinia + Electron，纯前端 localStorage，无后端；29 个主视图 + 47 个工具组件 + 游戏）
> - **当前项目** `work-system/mini-program`（uni-app 小程序前端）+ `work-system/server`（NestJS + TypeORM + MySQL 后端）
>
> 架构差异：原始为「每个功能一个专属富交互视图」；当前用「19 个专属页面 + 1 个通用 CRUD 页（`crud.vue` + `crud-schema.js` 的 29 个实体配置）」替代。
> 后端 29 个实体的通用 CRUD 已完整注册（`CrudController` 子类覆盖全部前缀），即**基础增删改查数据层可用**；缺口集中在**前端领域化交互缺失**与**部分模块完全无对应**。

## 二、详细缺口清单（按优先级）

### P0 · 核心流程缺口（教师每日必用，当前不可用/严重退化）

#### G-01 课表可视化展示缺失
- **所属模块**：课表（school/schedules）
- **当前完成度**：后端实体 `ScheduleItem` + 通用 CRUD 录入可用；toolbox 「课表」入口仅打开通用表单。
- **缺失部分**：无「周视图课程表」可视化页面（原始 `Schedule.vue` 271 行，按星期×节次渲染网格、显示科目/教师、单双周/早晚自习标色）。
- **与原项目差异**：原始为教师每日打开的网格课表；当前只能逐条录入记录，无法一眼看全周课表。
- **可执行项**：新增 `pages/schedule/schedule.vue`，从 `/schedules?classId=` 拉取并按 dayOfWeek×period 渲染周网格，复用 `weekType`/`section` 字段。

#### G-02 考勤网格交互缺失
- **所属模块**：考勤（attendances）
- **当前完成度**：后端 `attendances` 实体 + CRUD；表单可存一条 `records`(JSON)。
- **缺失部分**：无「学生列表×出勤状态」网格、无逐人点击切换（出勤/迟到/请假/旷课）、无「一键全班标记」、无实时统计卡片。
- **与原项目差异**：原始 `Attendance.vue` 194 行含 `statusMap` 配色、逐人 `setStatus`、全班 `setAll`、四态计数 `stats`。
- **可执行项**：新增 `pages/attendance/attendance.vue`，按班级加载学生，渲染状态网格并写回 `attendances.records`。

#### G-03 作业状态流转缺失
- **所属模块**：作业（homework）
- **当前完成度**：后端 `homework` 实体 + CRUD 表单（含 status 字段定义）。
- **缺失部分**：无「待批改/已批改/已发还」状态看板、无按状态筛选、无逐条「标记已批改」、无待批改/已完成计数。
- **与原项目差异**：原始 `Homework.vue` 534 行含 `statusFilters`、`pendingCount`/`doneCount`、`updateHomework` 批改流。
- **可执行项**：新增 `pages/homework/homework.vue`，列表+状态流转+统计；或扩展 crud 页支持状态快捷操作。

#### G-04 班级公告看板缺失
- **所属模块**：公告（notices）
- **当前完成度**：后端 `notices` 实体 + CRUD 表单（含 `pinned`/`ended` 字段）。
- **缺失部分**：无公告看板展示、无置顶切换 UI、无「隐藏已结束」筛选、无发布/可见范围逻辑。
- **与原项目差异**：原始 `Notice.vue` 318 行含 `filtered`(置顶优先+隐藏已结束)、`pinned` 切换。
- **可执行项**：新增 `pages/notice/notice.vue` 看板页，或扩展 crud 页支持置顶/筛选交互。

#### G-05 个人中心缺失
- **所属模块**：账户/设置（users）
- **当前完成度**：后端 `users` 实体 + `users.controller`（2 路由）、`auth` 登录可用；无个人信息/设置页。
- **缺失部分**：无「个人中心」页面（教师资料、修改密码、偏好设置、我的统计）。
- **与原项目差异**：原始 `Profile.vue` 659 行，含教师档案、设置、数据概览。
- **可执行项**：新增 `pages/profile/profile.vue`，调 `/users` 读取/更新当前教师信息。

### P1 · 核心分析 / 数据类缺口

#### G-06 成绩趋势图表缺失
- **所属模块**：成绩分析（grades + exams）
- **当前完成度**：成绩/考试 CRUD 可用；`ai-exam` 仅做单次考试 AI 分析。
- **缺失部分**：无跨考试「班级/科目平均分趋势折线图」、无「跨班对比」、无「学生个体成绩轨迹」。
- **与原项目差异**：原始 `GradeTrend.vue` 329 行含 `trendData`/`compareData`/`studentTrend` 三类图表数据（ECharts）。小程序端需引入图表库（如 uCharts/echarts-for-weixin）。
- **可执行项**：新增 `pages/grade-trend/grade-trend.vue`，聚合 exams×grades 计算趋势，渲染图表。

#### G-07 教学资源文件上传缺失
- **所属模块**：资源库（resources）
- **当前完成度**：后端 `resources` 实体 + CRUD 表单（仅 `url` 文本）。
- **缺失部分**：无本地文件/图片上传（原始 `Resource.vue` 用 `<input type=file>` 上传），仅能填外链 URL。
- **与原项目差异**：原始支持真实文件上传与本地存储；当前无上传通道。
- **可执行项**：后端加 `/resources/upload`（对象存储/云托管静态），前端 crud 页支持选文件上传回填 `url`。

## 三、空壳 / 风险点专门提示（对应需求第 2 条）

1. **通用 CRUD 页本身即「统一空壳」**：`crud.vue` 对 29 个实体提供一致的表格+表单，但**不含任何领域业务逻辑**（无状态流转、无统计、无可视化、无文件上传）。凡经 toolbox 入口进入的模块，其「富交互」全部丢失。
2. **后端 orphan**：`engagement` 模块无前端入口（G-17）。
3. **字段级空壳**：`attendances.records`、`duty-rosters.assignments`、`class-duty-configs.duties/assignments`、`class-activities.photos`、`resources.tags` 等 JSON 字段在 crud 页仅以 `textarea` 文本录入，无结构化编辑器，录入即「手写 JSON」，易错且无校验。
4. **课表 `section`/`weekType` 已建模但无消费端**：字段存在（G-01 缺展示页），数据写入后无可视化消费。
