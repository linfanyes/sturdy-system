# 园丁工作台 · 微信小程序 · 系统说明书

> 适用对象：一线教师 / 班主任
> 版本：小程序端（uni-app），配套 NestJS 后端 + 腾讯云数据库 MySQL 8.0
> 本文档说明**产品能做什么、怎么用、数据存在哪里**，不涉及部署（部署见《微信小程序前后端完整部署与发布流程》）。

---

## 一、产品简介

**园丁工作台**是面向教师的班级管理一站式工具。小程序版把桌面版的核心能力搬到了手机上：班级管理、学生档案、考试与成绩、座位与分组、AI 备课生成、18 款课堂小游戏，以及语文/数学/英语专项练习、文字办公等共 **53 个页面、44 类工具**，全部可在微信里随手使用。

**设计原则**
- 所有密钥（微信、AI）只在后端配置，手机端不存储，安全合规。
- 数据按教师隔离：你只看到自己班级的数据。
- 小游戏、口算、倒计时等工具**无需联网**即可使用；班级数据、AI 生成需联网并经后端。

## 二、系统架构（用户视角）

```
┌──────────────┐     联网（HTTPS）     ┌──────────────────┐     联网     ┌─────────────────┐
│  微信小程序   │ ───────────────────► │  后端服务（云托管） │ ─────────► │  云数据库 MySQL  │
│  （你的手机） │ ◄─────────────────── │  NestJS          │ ◄───────── │  （你的数据）    │
└──────────────┘                      └──────────────────┘            └─────────────────┘
```

- **你（老师）**：在微信里打开小程序，输入/查看班级数据，使用各类工具。
- **后端服务**：负责登录鉴权、保存数据、调用 AI 生成内容。由学校/管理员部署在「微信云托管」。
- **数据库**：你的班级、学生、成绩等数据存放在这里，按老师分库隔离。

> 你不需要懂上面这些也能用。只要管理员把后端和数据库部署好，并告诉你小程序的名字/二维码，你微信里打开即用。

## 二-B、数据模型与实体关系

小程序中**教师、班级、学生、考试、成绩**五个核心实体的关系如下：

```
教师 ←→ 班级（任教：一个教师教多个班，一个班有多位教师）
  │
  ├── 课表（schedules）：教师作为字符串字段存储（姓名），按姓名匹配显示
  │
班级 ←→ 学生（一对多：一个班级有多个学生）
  │
  ├── 考勤（attendances）：按 classId + date 记录，每条包含 { studentId, status }
  ├── 行为记录（behavior-records）：按 studentId 关联学生
  ├── 轮值表（duty-rosters）：按 classId 过滤
  ├── 班费（class-expenses）：按 classId 过滤
  ├── 班级活动（class-activities）：按 classId 过滤
  │
考试 ←→ 成绩（一对多：一次考试有多个科目的成绩）
  │
  └── 成绩单（grades）：每条含 classId + name + subject + scores[{ studentId, score }]
       └──→ 按 studentId 关联学生，按 classId 关联班级
```

**关系合理性说明**：
- ✅ 班级↔学生：通过 `classId` 外键关联，一对一从属关系，合理
- ✅ 考试↔成绩：`grades` 通过 `classId` 和 `studentId` 双向关联，支持跨班考试和单科成绩录入
- ✅ 学生↔考勤/行为记录：通过 `studentId` 关联，支持跨班级维度查询学生发展轨迹
- ⚠️ **教师↔课表**：`schedule.teacher` 为**字符串（教师姓名）**，非教师 ID。若教师改名或后端存储格式不一致，课表对应的教师名称会显示异常（已在「设置」页通过名称+工号兜底匹配缓解）
- ⚠️ **学生删除后的孤立数据**：`grades.scores[].studentId`、`attendances.records[].studentId`、`behavior-records.studentId` 在学生删除后不会自动清理（需后端兜底或数据库级联删除）

## 三、登录与账户

1. 首次打开小程序，点击「微信登录」（授权获取微信昵称/头像，仅用于标识账号）。
2. 系统用你的微信身份签发登录态（JWT），后续操作自动带上。
3. 登录态长期有效；若提示「请重新登录」，重新点一次微信登录即可。
4. **多设备**：同一微信账号在另一台手机登录，数据互通（数据存在云端，不在本机）。

> 适用于：未部署后端时展示 / 试用小程序全部功能。

小程序内置**演示模式**，可在「设置 → 演示模式」中一键开启：

| 功能 | 说明 |
| --- | --- |
| **开启** | 切换到设置页底部「演示模式」开关 → 开启 |
| **效果** | 所有 API 请求自动返回本地模拟数据（含示例班级、学生、成绩、课表等），无需真实后端即可浏览全部页面 |

| **AI** | 演示模式下 AI 也返回模拟回复，不必配置 AI 密钥 |

> 演示模式状态持久化到手机本地存储，下次打开自动恢复。适合向学校领导展示系统功能、或在新环境快速熟悉操作流程。

## 五、核心教学管理

### 5.1 班级管理
- 新建/编辑班级（年级、学科、班主任等），学生归属到班级。
- 班级列表默认按学号升序展示学生。

### 5.2 学生管理
- 学生档案：姓名、学号、所在学校/班级、座次号、职务、家长联系方式。
- 支持**导入**（Excel/模板）批量建学生；导入时若含家长信息，**自动同步到家长联系**。
- 学生详情展示座次与职务，便于课堂快速定位。

### 5.3 考试管理
- 新建考试：默认勾选语文/数学/英语/科学/品德，其他科目默认不勾选（不参与考试）。
- 每次考试可记录各科成绩，自动排名（显示全班）。

### 5.4 成绩管理
- 录入/导入成绩，按考试查看排名、均分、优秀率、及格率、分数段。
- 「考试一键分析」可调用 AI 自动生成「本次考试分析情况描述」，并回写到考试记录。

### 5.5 座位表
- 可视化网格排座，学生分布在讲台两侧；使用中座次框用醒目颜色区分。
- 支持按「第 X 行 · 第 Y 列」编码，与新增/导入学生的座位号编码一致。

### 5.6 随机分组
- 按人数、按列等多种方式随机分组，课堂活动快速分队。

### 5.7 我的课表（可视化周视图）
- 按「星期 × 节次」渲染周课表网格，显示科目 / 任课教师；复用 `单双周`(all/single/double) 与 `早读/晚自习` 等非数字节次。
- 支持从图片 / Excel 智能识别导入（后端 AI 解析 + 幂等落库）；也可手动逐格录入。

### 5.8 考勤管理（状态网格）
- 按班级加载学生名单，逐人点击切换 **出勤 / 迟到 / 请假 / 旷课** 四态（配色区分），支持「一键全班标记」。
- 顶部实时统计四态人数；每日记录写回 `attendances`。

### 5.9 作业管理（状态流转）
- 作业列表按 **待批改 / 已批改 / 已发还** 状态流转，支持按状态筛选与「标记已批改 / 已发还」。
- 顶部展示待批改、已完成计数，方便跟踪批改进度。

### 5.10 班级公告（看板）
- 公告看板展示，**置顶**公告优先显示，可切换「隐藏已结束」；发布 / 删除即时生效。

### 5.11 教学资源（含上传）
- 资源列表管理；支持**图片 base64 上传**（小程序私有链路下直存，无需对象存储）与外部链接两种形式，按分类 / 标签检索。

### 5.12 个人中心
- 查看与修改教师资料（姓名 / 学科 / 学校 / 学期 / 座右铭 / 头像），切换**深色 / 浅色主题**，退出登录。

### 5.13 成绩趋势
- 聚合 `exams × grades` 计算趋势：班级 / 科目**平均分趋势**（跨考试）、**跨班对比**、学生个体成绩轨迹；纯 CSS 柱状图渲染，无需引入图表库。

## 六、AI 智能生成（备课提效）

> 需管理员在后端「设置」页配置 AI 密钥与模型；未配置时页面会提示「未配置 AI 密钥」，不会崩溃。

- **知识点生成**：填年级/科目/教材/学期 + 主题 → 生成结构化讲解（定义/重难点/易错点/例题/复习建议）→ 可存入知识库。
- **优选试卷生成**：填年级/科目/年份/主题 → 生成贴近真实考纲的试卷（题目+答案+解析，模拟题明确标注）→ 存入试卷库。
- **考试一键分析**：选考试 → 前端汇总成绩统计 → AI 生成分析描述 → 回写考试记录。

## 十一、数据管理与教师行政

> 说明：本系统的数据层由统一的「通用 CRUD 控制器」支撑（后端 29 个实体全部注册、按教师隔离）。其中**高频核心模块已升级为专属可视化页面**（见第五节 5.7–5.13、第九/十节工具）；其余长尾模块在工具箱中以「通用表单」录入查询。

- **教学备课类（专属页）**：课表、考勤、作业、公告、资源、成绩趋势（见 5.7–5.13）。
- **教学备课类（CRUD）**：教案、教案模板、知识点、优选试卷、试卷查询、听课记录。
- **班级事务类（专属页）**：轮值表（排班网格）、班费（收支统计）、班级活动（图文流）、班级风采（相册）、个人中心。
- **班级事务类（CRUD）**：值日配置、待办、笔记、抽签历史。
- **学生发展类（专属页）**：成长记录（时间线）、家长联系（跟进管理）。
- **学生发展类（CRUD）**：行为记录、获奖记录、奖项类别、通知模板。
- **engagement 模块（加减分 / 积分记录 / 小组评分）**：三个实体已注册后端并提供 CRUD，工具箱中「加减分 / 积分记录 / 小组评分」即其前端入口，数据互通，并非孤儿模块。
- **教师行政类（专属页）**：教师通讯录（检索 / 星标 / 拨号）。
- **教师行政类（CRUD）**：工作日志。

> 以上数据均保存在云端数据库，按教师隔离；换手机登录同一微信账号数据不丢。

## 十二、数据存储与隐私

- **存在哪里**：班级、学生、成绩等业务数据存于后端云数据库；登录态（token）仅存于你本机小程序缓存。
- **密钥安全**：微信 AppSecret、AI 密钥**只在后端「设置」页配置**，手机端拿不到，降低泄露风险。
- **数据隔离**：系统用你的教师身份过滤数据，看不到其他老师的班级内容。
- **本地工具数据**：错题本、文案模板库保存在你本机（uni.storage），不上云、不跨设备；如需备份请手动导出或截图。

## 十三、常见问题（FAQ）

**Q1：打开小程序提示「网络错误 / 请求失败」？**
A：通常是后端地址未配置或域名未备案。请联系管理员确认 `API_BASE` 与微信公众平台「服务器域名」已正确设置（详见部署手册）。

**Q2：AI 工具点生成没反应 / 提示未配置？**
A：后端「设置」页还没填 AI 密钥或模型地址。请管理员配置后重试。

**Q3：为什么有些工具不用联网也能用？**
A：18 款小游戏、口算、倒计时、计算器、单位换算、乘法口诀、错题本、文案模板库是本地逻辑，专为课堂弱网/无网场景设计。

**Q4：我在 A 手机录入的数据，B 手机会有吗？**
A：联网录入的班级/学生/成绩等数据会同步（云端）；但错题本、文案模板库是本机存储，不跨设备。

**Q5：数据会丢吗？**
A：云端数据由腾讯云数据库自动备份（管理员开启）；本机数据（错题本/模板）需自行注意，换手机前建议导出。
