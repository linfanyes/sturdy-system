# 园丁工作台 (Gardener Workbench)

面向教师的一站式工作台：班级管理、学生档案、成绩分析、课表、笔记、待办、奖惩，并内置 AI 助理与一系列提效工具（优选试卷、优质教案、评语生成、翻译等）。所有数据保存在浏览器本地，开箱即用、无需后端。

> 项目标识：`trace`（package.json name）。界面与文档统一称「园丁工作台」。

## 技术栈

- **前端框架**：Vue 3（`<script setup>` SFC） + TypeScript
- **构建工具**：Vite 5
- **状态管理**：Pinia 2
- **路由**：Vue Router 4（hash 模式，`#/` 为工作台首页）
- **样式**：Tailwind CSS 3 + `clsx` / `tailwind-merge`
- **图标**：`lucide-vue-next`
- **AI / 文档解析**：`marked`（Markdown）、`docx`（Word 导出）、`pdfjs-dist` / `jszip` / `xlsx`（附件解析为文本）
- **其他**：`dayjs`（日期）、`nanoid`（ID）

## 快速开始

```bash
npm install        # 安装依赖
npm run dev        # 启动开发服务器（默认端口 5201 → http://localhost:5201）
npm run check      # 类型检查（vue-tsc -b）
npm run build      # 生产构建（vue-tsc -b && vite build → dist/）
npm run preview    # 预览构建产物
npm run lint       # ESLint 检查
```

> 端口在 `vite.config.ts` 中固定为 `5201`（`server.host: true`，允许局域网/本地域名访问）。构建使用相对路径 `base: './'`，产物可部署到任意子目录。

## 生成测试 / 演示数据

开发模式下可一键为**当前登录账号**填充一套完整的演示数据，便于快速体验各功能，无需手动录入。

**入口**：登录后（登录码 `1314520`）在地址栏访问 `#/dev/seed`（即 `http://localhost:5201/#/dev/seed`）。

> 该页面**仅在开发模式**（`npm run dev`）可用；生产构建（`npm run build`）不包含此入口。

**操作步骤**：
1. 进入页面后，点击「一键生成测试数据」按钮。
2. 工具会先**清空当前账号已有的**班级 / 学生 / 考试 / 成绩 / 课表 / 笔记 / 待办等数据，再生成全套新数据（避免重复叠加）。
3. 生成完成后点「回到首页」即可进入工作台查看效果。

**生成内容（默认）**：
- 2 个班级（三年级一班 / 二班，班主任均为「甘珊」），共 30 名学生
- 自动排课（周一至周五，每天 8 节）
- 3 次考试（第一单元测试 / 期中考试 / 第二单元测试），含各科满分（语数英 100、其余 50）
- 各科成绩（按学生姓名生成的稳定真实分数）
- 班级奖惩 8 条、课表资源 5 个、教师 11 位、班级 / 全校公告 5 条
- 笔记 4 篇（含 1 置顶 + 1 收藏）、待办 6 条（含 1 已完成）

**数据存储**：全部写入当前登录用户隔离的 `localStorage`（`trace.{userId}.*`）。如需在自己常用的浏览器中查看，请先登录对应账号再运行生成，或在生成所用的同一浏览器中访问工作台。

### AI 工具箱

- **AI 对话**：自动注入本系统「使用说明书」与你的班级 / 学生 / 成绩 / 课表 / 笔记 / 待办 / 奖惩等真实数据，回答基于真实数据，不编造；支持根据图片/作图意图自动切换多模态模型。
- **优选试卷**：按年级 / 学科 / 题型 / 难度等生成试卷，可下载 Markdown / Word / 打印。
- **优质教案**：AI 产出多套候选方案，勾选后整合为一篇教案。
- **评语生成**：基于学生本学期整体表现生成评语，并自动写入每位学生档案。
- **口算生成**：加减乘除自动出题；打印 / 下载 TXT 时顶部自动带学校、班级、出题教师信息。
- **翻译**。

> 每个工具页面右上角（或顶部）均有「返回工具箱」按钮，方便用完快速回到工具箱主页。

## AI 模型配置（默认已设为阿里通义千问）

进入「AI 对话 → 右上角「设置」」：

- **Base URL（默认）**：`https://dashscope.aliyuncs.com/compatible-mode/v1`
- **模型（默认）**：`qwen-plus`
- **可选多模态模型**：`qwen-vl-plus`（具备图片识别能力）
- **自动模型切换**：发送消息时，若当前消息含图片附件 / 会话历史含图片 / 检测到作图意图，会自动切换到多模态模型；无图片需求时自动切回文本模型，减少手动切换。
- 支持任意 OpenAI 兼容协议；模型名也可选「+ 自定义…」填写其他兼容模型。
- **附件提示**：文档类附件（PDF / Word / Excel / PPT / ODT）会自动解析为文本后发送，不依赖多模态模型。

> 若你此前已保存过 AI 设置（localStorage 中有旧配置），需手动在「设置」中切换，或点「恢复默认设置」应用上述新的阿里默认配置。
